void canix_SFP_RTS_handler ( const canix_frame *  frame  ) 

Stellt den HCAN RTS Frame Callback dar. Es werden Zeit-Informationen verarbeitet:

  • HCAN_RTS_TIME_INFO
  • HCAN_RTS_DATE_INFO

Diese werden vom Time-Master gesendet; jeder RTS Empfaenger synchronisiert sich dazu, sofern die Abweichung einen bestimmten Betrag ueberschreitet.

Hinweis: Das Datum wird immer uebernommen.

Es wird aber nicht die Time-Master Funktion uebernommen, wenn eine Weile keine Time-Meldungen mehr kommen. Dieses Feature ist z.Z. nur in der Controller-1612 Firmware implementiert.

Auch werden Zeit-Anfrage beantwortet:

  • HCAN_RTS_TIME_REQUEST
  • HCAN_RTS_DATE_REQUEST

Ueber diese Befehle koennen HCAN Clients die aktuelle Zeit / das Datum auslesen.

Definiert in Zeile 34 der Datei hcan_rts.c.

Benutzt canix_frame_send(), canix_rtc_clock, canix_selfaddr(), canix_frame::data, canix_rtc_clock_t::day_of_month, canix_rtc_clock_t::day_of_week, canix_frame::dst, canix_rtc_clock_t::hour, canix_rtc_clock_t::last_date_update, canix_rtc_clock_t::minute, canix_rtc_clock_t::month_of_year, canix_frame::proto, canix_rtc_clock_t::second, canix_frame::size, canix_frame::src und canix_rtc_clock_t::year.

Wird benutzt von canix_init().
